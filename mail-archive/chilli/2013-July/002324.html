Hi CoovaChilli List,

i have coovachilli_1.3.0 running as central server in multi
tenancy environment (ISP Data Center) using VLANs. CoocaChilli is serving
about 50 hotspots across the city. A separate CoovaChilli instance is
running on each VLAN. All instances work normal, without any issue. But i
have asked to provide separate landing page for each customer with their
company logo on the login page.

i have the following setup:

root at captive:/usr/local/etc/chilli# ls -l uploads/

drwxr-xr-x  7 www-data www-data 4096 Jul 29 16:04 ./
drwxr-xr-x 10 root     root     4096 Jul 25 15:06 ../
drwxrwxr-x  5 chilli   www-data 4096 Jul 29 15:24 www-cafeclub/
drwxrwxr-x  4 chilli   www-data 4096 Jul 29 16:17 www-yzone/

*nano /usr/local/etc/chilli/coova-vlan10/config*

HS_WWWDIR=/usr/local/etc/chilli/uploads/www-cafeclub
HS_WWWBIN=/usr/local/etc/chilli/upload/wwwsh
HS_UAMFORMAT=http://
\$HS_UAMLISTEN:\$HS_UAMUIPORT/uploads/www-cafeclub/login.chi
HS_UAMHOMEPAGE=http://
\$HS_UAMLISTEN:\$HS_UAMPORT/uploads/www-cafeclub/coova.html
*nano /usr/local/etc/chilli/coova-vlan20/config*

HS_WWWDIR=/usr/local/etc/chilli/uploads/www-yzone
HS_WWWBIN=/usr/local/etc/chilli/upload/wwwsh
HS_UAMFORMAT=http://
\$HS_UAMLISTEN:\$HS_UAMUIPORT/uploads/www-yzone/login.chi
HS_UAMHOMEPAGE=http://
\$HS_UAMLISTEN:\$HS_UAMPORT/uploads/www-yzone/coova.html
Firefox Reports: Firefox has detected that the server is redirecting the
request for this address in a way that will never complete.

*SOLUTION: *
i wish to reply my own post because i am able to solve the problem. Here is
the solution for following requirement.

# Phrase Included For Google Search From Archives
CoovaChilli Multi-Tenancy Running Multiple VLANs, Separate Login Page For
Each Customer / Client. Different Landing Page For Each Client / Customer,
Multiple Login Pages CoovaChilli

*root at captive:/usr/local/etc/chilli# ls -l uploads/*
drwxr-xr-x  7 www-data www-data 4096 Jul 29 16:04 ./
drwxr-xr-x 10 root     root     4096 Jul 25 15:06 ../
drwxrwxr-x  5 chilli   www-data 4096 Jul 29 15:24 www-cafeclub/
drwxrwxr-x  4 chilli   www-data 4096 Jul 29 16:17 www-yzone/

*nano /usr/local/etc/chilli/coova-vlan10/config*
HS_WWWDIR=/usr/local/etc/chilli/uploads/www-cafeclub
HS_WWWBIN=/usr/local/etc/chilli/wwwsh
HS_UAMFORMAT=http://\$HS_UAMLISTEN:\$HS_UAMUIPORT/www/login.chi
HS_UAMHOMEPAGE=http://\$HS_UAMLISTEN:\$HS_UAMPORT/www/coova.html
*
*
*nano /usr/local/etc/chilli/coova-vlan20/config*
HS_WWWDIR=/usr/local/etc/chilli/uploads/www-yzone
HS_WWWBIN=/usr/local/etc/chilli/wwwsh
HS_UAMFORMAT=http://\$HS_UAMLISTEN:\$HS_UAMUIPORT/www/login.chi
HS_UAMHOMEPAGE=http://\$HS_UAMLISTEN:\$HS_UAMPORT/www/coova.html

restart chilli daemon(s)
Please do not forget to check the permissions etc... on the "www"
directories
