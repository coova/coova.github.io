Hi everyone,

 

I've got a very strange problem.

 

Setup:

 

Chilli 1.2.9 Running with Layer3

 

Client with static IP provided by isc-dhcp running on VLAN ID 40

 

 

If I make the login normally (open browser, type user and password) the
internet works great.

 

If I do the login on the chillibox with:
/usr/local/sbin/chilli_query -s /usr/local/var/run/chilli.vlan40.sock login
ip 10.1.1.xx username <username> password <password>

 

The login is performed good (show the session on radius) and internet start
working, but the speed is terribly slow.

 

If I logout and do the login via web browser on client, the speed is good
again.

list:

1392508928 0 0%/1024000 0%/3072000

 

If I do it via web browser:

 

10.1.40.19 51f032b400000004 1 <USERNAME> 11/0 0/1800 129190/0 40875/0

(1Mb)

 

Is there any way that chilli_query if failing on the login process?
