Hi,

I don't need the "Administrative Login" feature but it seems that
there is no way to disable it: removing the adminuser and
adminpassword parameters or setting it to an empty string doesn't
work: it still sends an Access-Request and an Accounting-Request.
Also, there are default values for adminuser and adminpassword
(chillispot/chillispot). Because administrative login is not mandatory
(chilli can work without it), is it important to have these default
values?

I wrote a patch that remove default values for adminuser and
adminpassword and doesn't send the RADIUS requests if adminuser is not
set.

I hope it could help someone!

Cheers,
Mathieu Alengrin
