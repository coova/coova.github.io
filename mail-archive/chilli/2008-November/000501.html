I'm having a bit of a problem with the usage of chilli_query. What I want to do is manually authenticate an IP-adress. (For users who can't display the captive portal.) Seems to me that this is what chilli_query is built for.

chilli_query authorizes the users just fine, but the sessiontimeout argument doesn't seem to work for me. Any suggestions?

The commands I'm using are:

chilli_query authorize ip 10.1.0.2 sessiontimeout 60 username testuser
This allows the ip adress 10.1.0.2 access, however chilli_query list shows:
<macadress> 10.1.0.2 pass <sessionid> 1 testuser 5/0 5/0 <originalurl>

The number indicating the sessiontimeout is 0, and it should be 60, right? The session also doesn't get disconnected after 60 seconds.

The sessions which are authorized through the captive portal shows up with a correct non-zero sessiontimeout value in chilli_query list.

Any ideas?
